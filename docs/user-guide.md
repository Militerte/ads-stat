# Ads stat — инструкция для пользователя

Дополнение **Ads stat** загружает статистику из **Яндекс.Метрики** и **Яндекс.Директа** в **Google Таблицы**. Данные появляются на выбранном листе в виде таблицы.

Поддержка: **militerte@gmail.com**

---

## Что нужно заранее

1. Аккаунт **Google** и доступ к Google Таблицам.
2. Аккаунт **Яндекса** с доступом к нужным счётчикам Метрики и/или рекламным кампаниям в Директе.
3. Установленное дополнение **Ads stat** (из Google Workspace Marketplace или по ссылке от администратора).

Для отчётов **Директа** у вашего аккаунта Яндекса должен быть доступ к API Директа (обычно это настраивает владелец рекламного кабинета).

---

## Установка и первый запуск

### 1. Установите дополнение

1. Откройте [Google Workspace Marketplace](https://workspace.google.com/marketplace) и найдите **Ads stat**, либо перейдите по ссылке установки.
2. Нажмите **Установить** и подтвердите доступ Google к таблице (разрешения запрашивает Google от имени дополнения).

### 2. Откройте панель в таблице

1. Откройте или создайте **Google Таблицу**.
2. Меню **Расширения** → **Ads stat** (или **Дополнения** → Ads stat).
3. Справа откроется **боковая панель** дополнения.

### 3. Войдите в Яндекс

1. Нажмите **Новый отчёт** (или откройте существующий пресет).
2. В блоке авторизации нажмите **Войти через Яндекс**.
3. Подтвердите вход в аккаунте Яндекса в открывшемся окне.
4. После успешного входа отобразится ваш логин Яндекса.

Кнопки в блоке аккаунта:

| Кнопка | Действие |
|--------|----------|
| **Проверить вход** | Проверить, что сессия Яндекса ещё действует |
| **Сменить** | Войти под другим аккаунтом Яндекса |
| **Выйти** | Отключить текущий аккаунт Яндекса в этом дополнении |

## Главный экран «Отчёты»

На главном экране вы видите:

- **Ежедневное обновление** — автоматическая перезагрузка всех сохранённых пресетов по расписанию.
- **Новый отчёт** — создать и настроить новую выгрузку.
- **Сохранённые пресеты** — список ранее сохранённых настроек для этой таблицы.

У каждого сохранённого пресета есть кнопки:

| Иконка | Действие |
|--------|----------|
| ▶ | **Запустить** — загрузить отчёт сейчас |
| ✎ | **Изменить** — открыть настройки пресета |
| ⧉ | **Копировать** — создать копию пресета |
| 🗑 | **Удалить** — удалить пресет |

Фильтр над списком (если есть пресеты) позволяет показать только пресеты для выбранного листа.

## Создание нового отчёта

1. Нажмите **Новый отчёт**.
2. Выберите источник:
   - **Яндекс.Метрика** — веб-аналитика, счётчики, UTM.
   - **Яндекс.Директ** — рекламная статистика (кампании, показы, клики, расход).

Далее настройте отчёт и нажмите **Загрузить сейчас** или **Сохранить пресет**.

---

## Отчёт из Яндекс.Метрики

### Счётчик

1. Раскройте блок **Счётчик**.
2. Нажмите иконку **обновления** (↻) справа — загрузится список счётчиков вашего аккаунта.
3. При необходимости введите поиск по названию или ID.
4. Выберите нужный счётчик из списка.

### Срез

В поле **Срез** доступны варианты:

- **По дням** — метрики по датам (удобно для динамики за период).
- **Произвольный срез** — вы сами выбираете группировки и метрики.

Для **произвольного среза**:

1. Добавьте **группировки** (необязательно) — кнопка **+** рядом со списком.
2. Добавьте **метрики** — обязательно хотя бы одну.
3. Без группировок отчёт будет одной строкой с итогами за период.

### Фильтры UTM

В блоке **Фильтры UTM** можно отфильтровать данные по меткам `source`, `medium`, `campaign`, `content`:

- несколько значений через **запятую** означает «**ИЛИ**»;
- для каждого поля можно выбрать условие: **точно**, **содержит**, **не равно**.

Кнопка **Показать фильтр** — посмотреть, как фильтр будет передан в API.

### Сортировка

Поле **Сортировка** влияет на порядок строк в отчёте (для среза «По дням» и произвольного).

## Отчёт из Яндекс.Директа

1. Выберите **группировки** (необязательно): день, кампания, группа объявлений, объявление и т.д.
2. Выберите **метрики**: показы, клики, расход.
3. При необходимости укажите **Фильтр: ID кампаний** — один или несколько ID через запятую. Если поле пустое, подтягиваются все доступные кампании.

Без группировок — одна строка с итогами за выбранный период.

## Период, лист и куда писать данные

Эти настройки общие для Метрики и Директа.

### Период

- **Последние N дней** — например, 30 дней назад от сегодня. При ежедневном автообновлении период **пересчитывается** каждый раз заново.
- **Фиксированные даты** — жёсткий интервал «с» и «по».
- **Исключить сегодня** — конец периода будет **вчера** (удобно, если данные за текущий день ещё неполные).

### Куда записать результат

| Поле | Описание |
|------|----------|
| **Лист** | Целевой лист таблицы или «активный лист» |
| **Ячейка** | Верхний левый угол таблицы с данными, например `A1` |
| **Отображать заголовки** | Первая строка — названия колонок |

Данные заполняются **вниз и вправо** от указанной ячейки. Существующие данные в этой области будут **перезаписаны**.

### Загрузка и сохранение

- **Загрузить сейчас** — один раз выгрузить отчёт с текущими настройками.
- **Сохранить пресет** — сохранить настройки в этой таблице под именем из поля **Название пресета**. Потом пресет можно запускать с главного экрана.

## Ежедневное автообновление

На главном экране в блоке **Ежедневное обновление**:

1. Включите галочку **Включить ежедневное автообновление**.
2. Укажите **время** (по часовому поясу Москвы).
3. Нажмите иконку **сохранения** (дискета) рядом с временем.

После этого в указанное время дополнение **автоматически перезагрузит все сохранённые пресеты** в этой таблице.

Иконка **обновления** (↻) рядом с временем — **обновить все пресеты прямо сейчас**, не дожидаясь расписания.

> Для автообновления Google должен иметь разрешение запускать дополнение без вашего присутствия — оно запрашивается при установке.

## Типичные сценарии

### Ежедневный отчёт по сайту

1. Метрика → счётчик → срез **По дням** → последние 30 дней → лист «Отчёт» → ячейка `A1`.
2. **Сохранить пресет** с понятным именем.
3. Включить **ежедневное автообновление** в 06:00.

### Статистика кампаний Директа

1. Директ → группировки: **День**, **Название кампании** → метрики: показы, клики, расход.
2. При необходимости — фильтр по ID кампаний.
3. **Загрузить сейчас** или сохранить пресет.

### Несколько отчётов в одной таблице

Сохраните несколько пресетов с разными именами и разными ячейками (например `A1`, `A50`, другой лист). Все они могут обновляться по одному расписанию.

## Частые вопросы и проблемы

### «Войдите в Яндекс» / не загружаются данные

Нажмите **Войти через Яндекс** и завершите авторизацию. Если не помогает — **Выйти**, затем войти снова.

### Пустой список счётчиков

Нажмите ↻ у блока **Счётчик**. Убедитесь, что у аккаунта Яндекса есть доступ к нужным счётчикам в Метрике.

### Ошибка при загрузке Директа

Проверьте, что у аккаунта есть доступ к рекламным кампаниям и API Директа. ID кампаний в фильтре должны быть корректными.

### Данные не обновились по расписанию

- Проверьте, что автообновление **включено** и время **сохранено** (иконка дискеты).
- Таблица и дополнение должны оставаться доступны; при длительном простое Google может откладывать триггеры.
- Запустите все пресеты вручную кнопкой ↻ на главном экране.

### Дополнение не открывается после установки

Обновите страницу таблицы (`Ctrl+Shift+R`). Закройте и снова откройте боковую панель через меню **Расширения**.

### Сообщение о том, что приложение не проверено Google

На этапе тестирования дополнения разработчик должен добавить ваш Google-email в список тестировщиков. Напишите на **militerte@gmail.com**.

- Отчёты записываются **только в вашу таблицу**.

- Отозвать доступ к Яндексу: кнопка **Выйти** в панели или настройки приложений на [oauth.yandex.ru](https://oauth.yandex.ru/).
